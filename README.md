## 📋 Overview
Project `ollama-vs-agent` is a technical deep-dive into the "Agentic" capabilities of local LLM orchestration. By integrating **Continue.dev** with local Ollama endpoints, we enable a zero-cost, privacy-first software development environment that rivals hosted solutions like GitHub Copilot or Claude Dev.

## 🏗️ Hardware Architecture
This session was benchmarked on **worlock**, a dual-NVIDIA system:
- **GPU 0**: NVIDIA RTX 3080 (10GB) - UI, Autocomplete, and Reasoning Offload.
- **GPU 1**: NVIDIA RTX 3060 (12GB) - Dedicated LLM Runner for 14B+ models.
- **Model Splitting**: Highly-complex 32B models are split across both VRAM pools (~20GB total utilization).

## 🚀 Key Projects

### 1. Codebase Health Monitor (CHM)
A persistent scanner located in `/chm_project` that identifies technical debt and code duplication.
- **Persistence**: SQLite backend (`chm_health.db`) stores scan timestamps, LOC, and duplication metrics.
- **Robustness**: Specially tuned to handle macOS resource forks (`._` files) and encoding anomalies.
- **Dashboard**: A standalone `dashboard.py` (generated by the 14B model) providing ASCII-based health summaries.

### 2. Research Summary (Branch: `research-summary`)
Automated extraction and formal refactoring of AI/ML papers.
- **Workflow**: Randomly selects PDFs from a local library -> Extracts via `pdftotext` -> Refactors via `32B` model.
- **Formats**: Core insights are refactored into:
  - **Symbolic Logic** (Predicate Calculus)
  - **Lambda Calculus** (Church Encodings)
  - **Haskell** (Functional Specifications)
  - **C++** (Template Metaprogramming)

## 🛠️ Usage
1. Configure `~/.continue/config.json` to point to local Ollama endpoints.
2. Ensure `ollama` is running with `OLLAMA_KEEP_ALIVE=-1` for dedicated sessions.
3. Use the `research-summary` branch to view architectural refactorings of technical papers.
